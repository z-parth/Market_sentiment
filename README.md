# Market Sentiment Analysis

A data analysis project exploring whether Bitcoin market sentiment actually affects how traders perform — and if so, how.

---

## What This Is About

I've always been curious about whether the Fear & Greed Index is just a fun metric people post on Twitter, or whether it genuinely reflects something real in how traders behave and make money. This project tries to answer that by combining the Bitcoin Fear & Greed Index with real trade-level data from Hyperliquid.

The core idea is simple: take daily sentiment scores going back to 2018, match them against actual trade outcomes from late 2024 onwards, and see what patterns emerge. Do traders make more money when the market is fearful? Are the people who go against the crowd actually more profitable? Does what happened yesterday in sentiment tell us anything about today's PnL?

These are the kinds of questions I wanted to dig into.

## The Data

### Bitcoin Fear & Greed Index
Daily sentiment scores from [alternative.me](https://alternative.me/crypto/fear-and-greed-index/) ranging from 0 (Extreme Fear) to 100 (Extreme Greed), with a classification label for each day. The dataset covers February 2018 through 2025, which gives a solid view across multiple market cycles.

| Column | Description |
|---|---|
| `timestamp` | Unix epoch timestamp |
| `value` | Sentiment score (0-100) |
| `classification` | Extreme Fear / Fear / Neutral / Greed / Extreme Greed |
| `date` | Calendar date |

### Hyperliquid Trader Data
Fill-level trade records from Hyperliquid covering December 2024 onwards. One thing worth knowing is that each row in the raw data is a partial fill — a single order can span dozens of rows — so a big part of the preprocessing work was aggregating these up to something meaningful.

| Column | Description |
|---|---|
| `Account` | Wallet address |
| `Coin` | Perpetual contract identifier |
| `Execution Price` | Fill price |
| `Size USD` | Trade size in USD |
| `Side` | BUY / SELL |
| `Timestamp IST` | Trade datetime (IST) |
| `Closed PnL` | Realised PnL on closing fills |
| `Fee` | Trading fee paid |

## Notebooks Walkthrough

### 01 — Data Loading & Merging
This is mostly plumbing — loading both files, parsing the timestamps (the IST format needed some careful handling), aggregating fill-level trades into daily summaries per account, and then joining everything on date. The output is a single merged dataframe saved to `data/processed/`.

### 02 — Cleaning & Preprocessing
The raw data has some pretty extreme outliers, especially in PnL and volume, so this notebook handles that with IQR-based flagging and winsorisation. The more interesting part is the feature engineering — I built out 16 new columns covering things like rolling sentiment averages, day-over-day sentiment change, a contrarian trade flag, directional bias per account, and a per-account profiling system that classifies traders into Losing, Break-even, and Profitable tiers.

### 03 — Sentiment vs PnL Analysis
The main analysis notebook. Works through six research questions with proper statistical testing (Kruskal-Wallis for group comparisons, Pearson correlation for the lag analysis). The last cell prints a clean summary of all the key findings automatically from the computed results.

### 04 — Visualization
Five dashboard-style figures that pull together everything from notebook 03 into something presentable. I used AI assistance to help design and refine the layout and styling of these dashboards — the goal was to go beyond basic matplotlib defaults and produce figures that actually look good and are easy to read at a glance. The full timeline dashboard in particular (Dashboard 5) combines sentiment, cumulative PnL, and volume into a single three-panel view that I think tells the story pretty clearly.

## Features Engineered

| Feature | Description |
|---|---|
| `fg_7d_ma` | 7-day rolling average of sentiment score |
| `fg_14d_ma` | 14-day rolling average of sentiment score |
| `fg_delta` | Day-over-day change in sentiment |
| `fg_momentum` | Direction of 3-day sentiment change (-1 / 0 / +1) |
| `is_extreme_fear` | Flag: FG score <= 20 |
| `is_extreme_greed` | Flag: FG score >= 80 |
| `is_contrarian` | Buying in Fear or selling in Greed |
| `bias` | Bullish / Neutral / Bearish based on buy ratio |
| `pnl_per_trade` | Net PnL divided by trade count |
| `avg_trade_size` | Average USD size per fill |
| `log_volume` | Log-transformed total volume |
| `trader_tier` | Account-level: Losing / Break-even / Profitable |
| `acct_win_rate` | Overall historical win rate of the account |
| `acct_contrarian_rate` | Share of days the account traded contrarian |

## Setup

**1. Clone the repo**
```bash
git clone https://github.com/YOUR_USERNAME/Market_sentiment.git
cd Market_sentiment
```

**2. Install dependencies**
```bash
pip install -r requirements.txt
```

**3. Add your raw data**

The raw CSVs are gitignored so you will need to drop them into `data/raw/` yourself:
```
data/raw/fear_greed.csv
data/raw/trader_data.csv
```

**4. Run notebooks in order**
```
01 -> 02 -> 03 -> 04
```

Each notebook picks up from where the previous one left off via the files in `data/processed/`.
